# MPU6050-I2C-to-SSD1306-SPI-Register-Control

본 프로젝트는 외부 라이브러리(`Wire.h`, `SPI.h`, Adafruit 드라이버 등)에 일절 의존하지 않고, 데이터시트 분석을 통해 ATmega328P의 하드웨어 레지스터를 로우 레벨에서 직접 제어하여 **MPU6050(I2C) 센서 데이터를 수확하고 SSD1306 OLED(SPI)에 실시간 렌더링하는 통합 시스템**입니다.

이전 프로젝트인 [MPU6050-I2C-to-USART-Register-Control]에서 다루었던 I2C 버스트 리드(Burst Read) 기반 위에, 이번에는 **SPI 통신 프로토콜과 디스플레이 드라이버 IC(SSD1306)의 하드웨어 타이밍 제어**를 새롭게 확장했습니다.

단순히 통신 인터페이스를 활성화하는 것에 그치지 않고, SSD1306 디스플레이 모듈의 내부 레지스터 구조와 Power ON/OFF 시퀀스, 전압 안정화 시간($t_{AF}$) 및 방전 대기 시간 등을 물리적 수준에서 정밀하게 분석했습니다. 이를 통해 매번 컴퓨터(PC 터미널)를 켜서 시리얼 모니터를 연결하지 않아도 전원만 넣으면 가속도 데이터 변화를 직관적으로 확인할 수 있으며, 토글 스위치를 통해 하드웨어 파손 없이 안전하게 디스플레이를 켜고 끌 수 있습니다.

MPU6050 및 SSD1306의 상세한 데이터시트 분석, 베어메탈 레벨의 통신 뼈대 설계 과정, 그리고 나노초($\text{ns}$) 단위의 하드웨어 클럭 타이밍 및 전압 안정화 시간($t_{AF}$)/잔류 전압 방전 시퀀스 분석 등 깊이 있는 로우레벨 디테일은 제 블로그에 상세히 기록해 두었습니다. 코드 너머의 물리적 동작 원리가 궁금하시다면 아래 링크를 참고해 주세요.

## SSD1306 SPI 통신 모드(Mode 0) 규격 도출 근거

외부 추상화 라이브러리 없이 SSD1306 드라이버 IC와 오차 없는 고속 동기화 통신을 수행하기 위해, 데이터시트의 타이밍 파형(Timing Diagram)을 분석하여 ATmega328P의 SPI 모듈 설정 규격을 도출했습니다.

* **SPI Mode 0 (CPOL = 0, CPHA = 0) 선정 근거**

<img width="605" height="270" alt="image" src="https://github.com/user-attachments/assets/f33c9dd8-ad1c-4ea6-9435-533cc3c07e3a" />

  * **CPOL = 0 (Clock Polarity):** 칩 선택 신호(`CS#`)가 비활성화(High) 상태이거나 통신 대기(Idle) 상태일 때, 직렬 클럭 라인(`SCLK`)이 **Low 레벨을 유지**하는 파형을 확인했습니다. 이에 따라 클럭의 기본 활성 레벨을 Low로 정의하는 `CPOL = 0`을 적용했습니다.
  * **CPHA = 0 (Clock Phase):** `CS#`가 Low로 떨어지며 통신이 시작된 후, 전송되는 직렬 데이터가 첫 번째 클럭의 상승 에지(Rising Edge)에서 정확히 샘플링(Sample, 1비트 전송)**되고, 하강 에지(Falling Edge)에서 다음 비트를 전선에 미리 올려두는 셋업(Setup) 동작을 수행하는 메커니즘을 규명했습니다. 따라서 첫 번째 에지에서 데이터를 수집하는 `CPHA = 0`을 도출했습니다.
  * 결론적으로 두 타이밍 조건이 결합된 **SPI Mode 0** 환경에서 MCU와 디스플레이 드라이버 간의 완벽한 하드웨어 동기화가 이루어집니다.

* **데이터 전송 방향 (MSB First):**
  * 타이밍 파형 하단의 데이터 주입 시퀀스를 분석한 결과, 직렬 데이터 입력 라인(`SDIN`)으로 데이터가 밀려 들어갈 때 **최상위 비트(D7)가 가장 먼저 주입**되고, 마지막 8번째 클럭에 최하위 비트(D0)가 적재되는 흐름을 확인했습니다. 이에 따라 MCU에서 데이터를 송출할 때 반드시 **MSB 우선 전송(MSB First)** 으로 제어해야 함을 명확히 정립했습니다.

## SSD1306 하드웨어 파워 시퀀스 및 타이밍 분석

디스플레이 모듈은 단순히 전원만 인가한다고 스스로 깨어나지 않으며, 종료 시 전원을 무작정 끊어버리면 패널에 맺힌 고전압 전하로 인해 소자가 파손될 위험이 있습니다. 데이터시트가 강력하게 경고하는 엄격한 하드웨어 전원 타이밍 명세를 분석하여, 안전한 파워 On/Off 시퀀스를 코드로 제어했습니다.

* **Power On 시퀀스 (내부 차지 펌프 승압 및 화면 출력):**

<img width="936" height="461" alt="image" src="https://github.com/user-attachments/assets/48c3bffb-42c4-4c99-9bc6-1514a07f83c6" />

  * **하드웨어 리셋 (t1 구간):** 내부 회로 전원(VDD) 인가 후, 리셋 핀(`RES#`)의 전압을 최소 유지 시간(3$\mu\text{s}$)보다 넉넉하게 10ms 동안 Low로 떨어뜨려 칩 내부 레지스터를 완전히 공장 초기화 상태로 고정합니다. 이 순간 패널 출력이 차단된 Display OFF 상태로 안전하게 진입합니다.
  * **내부 승압 회로 가동 및 전압 안정화:** 초기화 직후 차지 펌프 설정 명령어(`0x8D`, `0x14`)를 주입합니다. OLED 구동에 필요한 고전압($V_{CC}$)을 만들기 위해 승압 회로를 켜고, 전압이 단단하게 안착할 때까지 10ms 동안 물리적인 대기 시간을 가집니다.
  * **최종 화면 ON 및 $t_{AF}$ 대기:** 전압이 안정된 후 최종 화면 활성화 명령어(`0xAF`)를 송신합니다. 데이터시트 파형에 따르면 **명령을 보낸 직후 화면이 즉각 켜지는 것이 아니라 $t_{AF}$ 시간 이후에 픽셀 발광이 시작**되므로, 이를 고려해 100ms의 대기 시간을 주어 디스플레이가 완벽하게 켜지도록 보장합니다.

* **Power Off 시퀀스 (잔류 전하 방전 및 소자 보호):**

<img width="901" height="227" alt="image" src="https://github.com/user-attachments/assets/79a237b5-ce0a-4c60-a451-4df07cbf15d8" />

  * **구동 전류 차단:** 시스템 종료(스위치 토글) 시, 디스플레이 OFF 명령어(`0xAE`)를 먼저 송신하여 패널 유리에 인가되던 구동 전류를 즉각 차단하고 잔류 전하들이 방전되도록 유도합니다.
  * **승압 회로 비활성화:** 직후 차지 펌프 셧다운 명령어(`0x8D`, `0x10`)를 연속으로 날려 고전압을 생성하던 승압 회로 작동을 멈춥니다.
  * **잔류 전압 소멸 대기 (방전):** 고전압 라인($V_{CC}$)이 완전히 방전되기 전에 로직 전원(VDD)이 꺼지면 소자 수명이 단축되거나 역전류가 흐를 수 있습니다. 이를 막기 위해 **최소 100ms 이상의 물리적 딜레이**를 주어 내부 잔류 전압이 완전히 0V로 떨어질 때까지 기다리는 안전 셧다운 시스템을 구현했습니다.

## 결론 및 느낀점
이번 프로젝트는 이전 프로젝트들에 비해서 시간이 조금 더 걸렸었습니다. 데이터시트에서 타이밍과 회로도를 분석하는 건 많이 해왔었으니깐 쉬웠는데 이번 SSD1306 데이터시트는 이전의 다른 센서들과 조금 형식이 달랐으며 무엇보다 OLED 명령어와 모듈에 보내는 명령어가 각각 따로 있어서 전체적인 흐름과 모드들을 파악하는데 시간이 오래걸렸습니다.
그치만 항상 그랬듯이 해내고 나니깐 뿌듯하고 좋았습니다. 이젠 컴퓨터를 키지 않고도 센서 값을 전원만 연결해주면 확인이 가능하기 때문에 종종 자주 더 켜보는 것 같습니다. 감사합니다.
